Overview

The following seven use cases represent the most common tasks that the users are expected to perform. The first five, which involve creating, finding, leaving, and checking in to a game, are the core features of our application because they are what facilitate pickup sports games. The last two, log in and sign up, are essential for users to begin using the service.

Failure-handling is represented in the alternate flow of these use cases.

Find a Game

Brief Description

The user selects a game by querying the game scheduler. If no games match the query, the users is prompted to search again.

Preconditions

The user is logged on to the system.

Basic Flow

  1. Users selects the "Find game" screen
  2. User enters a kind of game
  3. The system presents a list of games of the desired kind
  4. 2 and 3 may be repeated
  6. The user selects a game
  7. The game scheduler registers the user with a game

Alternate Flow: Games not found

  1. If at 2 in Find a Game, no games are found
  2. The system displays the message "0 games found"
  3. The use case starts at 2) in find a game

Postcondition

The user is registered with a game

Leave a Game

Brief Description

The user leaves a game by updating the game schedule

Preconditions

The user is logged on to the system.

Basic Flow

  1. User selects the game they wish to leave on the main map
  2. The system presents game information with an option to leave
  3. The User selects the leave game option
  4. The game scheduler unregisters the user with the game
  6. 5) The user selects a game
  7. 6) The game scheduler registers the user with a game

Postcondition

The user is no longer registered with a selected game game

Create a Game

Brief Description

The user creates a game by updating the game scheduler. If invalid game information is provided, the users is prompted to enter information again.

Preconditions

The user is logged on to the system.

Basic Flow

  1. The user selects the "create game" screen
  2. User enters game information
  3. The user registers the game

Alternate Flow: Games not found

  1. If at 2) in create a game, some information is invalid:
  2. The system displays an error message
  3. The use case restarts at 2) in create a game

Postcondition

A new game is registered in the game scheduler

Signup

Brief Description

The user registers a new profile with the profile manager. This occurs during the first use of the app on a new phone via Facebook

Preconditions

The user has not logged on to the system via facebook before..

Basic Flow

  1. The user selects the log in screen
  2. The user input a username and password
  3. The user is prompted via facebook to confirm login credentials
  4. The system adds a new user profile with the information

Postcondition

The profile manager is updated with a new profile

Login

Brief Description

Preconditions

Basic Flow

  2. The system registers the user as logged with their device
