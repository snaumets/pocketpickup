Postmortem

Features

The 1.0 release of Pocket Pickup contatins the following features.

  • Login through Facebook
  • Create games (basketball, football, ultimate, softball, soccer, baseball)
  • Find games, based on time, location, distance radius and sport
  • Join games
  • Leave games
  • Delete games (if you're the only attendee
  • Set preferred sports, which will be used when finding games

Cuts

The main victims of our condensed schedule were:

  • Any sort of group functionality - this would have been a huge effort in the last week, so we decided to focus on testing and cleaning up other features instead
  • List view of games - we eliminated this since it duplicated functionality we already had. It'd still be a good idea in a final version of this app
  • Extended Facebook integration - cut due to time
  • Autoloading of pins - another time constraint cut
We saved at least 10 developer days by cutting out these features.

Roles

Serge: Project manager and lead developer.
Serge was our most productive developer. He was responsible for most of the Android-side UI elements and use cases.

Jacob: Secondary parse developer, documenter.
Jacob worked with a lot of the Parse backend code, and was responsible for a lot of the early documentation and subsequent edits. He also set up the Jenkins automatic build system.

Isaiah: Parse master.
Isaiah set up most of the functions to convert Android-facing objects into Parse data, and to retrieve that data in a format that the Android activities could use.

Patrick: Testing guru, front-end android developer.
Patrick started out working mainly with Android, but as the need for UI work tapered off, he took over unit/integration testing, and also set up our code coverage tests.

Viktor: Usability and stress tester, bug researcher.
Viktor spent most of his time testing our use cases in both normal and unusal ways. He would then help the appropriate developer figure out how to fix that bug.

Kane: Facebook integration developer, bug fixer and organizer.
I started out developing the facebook login feature, but once that turned out to be a short-term job, I ended up fixing UI bugs and cleaning up/refactoring our doumentation strategies.

Recap

Our roles stayed more or less as expected. One major change was shifting from each developer testing his own code to using Patrick as a dedicated tester. This was primarily due to the up-front time cost of learning the testing frameworks. It made more sense for one developer to sink that time and keep testing, rather than to get all of us on the same testing-strategy page. Our time management worked out well. Rather than adding features as fast as we could, we spent time polishing each feature we added. While this resulted in the cuts described above, we ended up with all features in a complete and bug-free state. We're comfortable with this tradeoff.
